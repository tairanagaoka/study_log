リポジトリとブランチの概念について学んだ。
①リポジトリについて
今までは「ブランチ = ソースの全量」というイメージだった
実際には リポジトリ = コミットの集まり
例：ABCリポジトリがあった場合、
Acommit
Bcommit
Ccommit

②ブランチについて
同じリポジトリ内で作業の分岐を作る機能
別作業を他の作業に影響させずに進められる
別タスクを同時に進められることが 最大の利点

💡補足メモ
将来的に複数ブランチから main にマージする際、同じ箇所を編集しているとコンフリクトが発生する
長期間作業する場合は、定期的に main の最新状態を取り込むと安全

今日使った、Gitコマンド

クローン(URLからローカルにコピー)
$ git clone https://github.com/tairanagaoka/study_log.git
↓
vscodeで編集後にステージングする(コミット用意する。)
git add .
↓
編集＆ステージングされた内容を確認
git status
↓
Githubにコミットするため、ユーザと mailを設定。
$ git config --global user.name "Taira Nagaoka"
$ git config --global user.email "tairanagaoka035@gmail.com"
↓
GitHubにコミット
git commit -m "Add Study_plan"
